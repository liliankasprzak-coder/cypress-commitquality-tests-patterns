CENÁRIO 1: Validar layout da tela de adicionar produto
========================================
Feature: Validação de Layout
Como analista de QA
Quero validar os elementos da página Add Product
Para garantir que todos os campos estão presentes

Dado que estou na página Add Product
Quando a página carrega
Então devo ver o campo "Product Name"
E devo ver o campo "Price"
E devo ver o campo "Date Stocked"
E devo ver o botão "Submit"
E devo ver o botão "Cancel"

Pattern Aplicado: Page Object Model
Benefício: Centraliza os seletores e ações da página

CENÁRIO 2: Adicionar produto completo com sucesso
========================================
Feature: Cadastro de Produto
Como usuário

Para adicionar ao sistema

Dado que estou na página Add Product
Quando preencho o nome do produto com dados gerados
E preencho o preço com dados gerados
E preencho a data com a data de hoje
E clico no botão Submit
Então o produto deve ser cadastrado com sucesso

Pattern Aplicado: Builder Pattern
Benefício: Facilita a criação de objetos complexos com dados dinâmicos

========================================
VANTAGENS DO USO DE DESIGN PATTERNS
========================================

1. Page Object Model:
   - Centralização dos seletores
   - Reutilização de código
   - Manutenção facilitada
   - Testes mais legíveis

2. Builder Pattern:
   - Criação flexível de objetos
   - Dados de teste consistentes
   - Facilita testes com dados variados
   - Código mais expressivo

3. Combinação dos Patterns:
   - Separação de responsabilidades
   - Testes mais profissionais
   - Fácil adaptação a mudanças
   - Redução de duplicação de código
